Dan ini adalah Roadmap kita:

FASE 0 — FOUNDATION (SEKALI SEUMUR HIDUP)
    Tujuan: frontend bisa “hidup” sebagai platform

    SESSION 0 — Bootstrap & Discipline
        - Vite + React + TS + Tailwind
        - Path alias (@/)
        - Strict TypeScript
        - Folder core / business / vertical
        - ESLint + Prettier

        Output:
        - Proyek stabil
        - Struktur tidak berubah lagi


FASE 1 — PLATFORM ACCESS (AUTH & SECURITY)
    Tujuan: user masuk sistem dengan benar

    SESSION 1 — Authentication & Entry
        - Login page
        - Auth service (token)
        - Session store
        - Permission store
        - Redirect logic

        Output:
        - Login → Dashboard
        - Permission aktif


    SESSION 2 — Routing & Guard
        - PrivateRoute
        - Role / permission guard
        - Redirect unauthorized

        Output:
        - Tidak bisa buka page tanpa izin


    SESSION 3 — Layout & Navigation
        - AppLayout
        - Sidebar
        - Topbar
        - Dynamic menu (permission-based)

        Output:
        - Struktur UI ERP permanen


FASE 2 — BUSINESS UI ENGINE
    Tujuan: frontend tidak hardcode domain bisnis

    SESSION 4 — Entity List (READ)
        - Generic table renderer
        - Column schema
        - Pagination
        - Sorting
        - Permission action (view)

        Output:
        - Semua entity bisa ditampilkan

    SESSION 5 — Entity Action (CRUD)
        - Row action
        - Bulk action
        - Permission guard button
        - Confirm dialog

        Output:
        - UI action aman & konsisten

    SESSION 6 — Form Engine
        - Dynamic form renderer
        - Field schema
        - Validation dari backend
        - Reusable input

        Output:
        - Form universal (product, customer, dll)

    SESSION 7 — Workflow UI
        - Status badge
        - Workflow action renderer
        - Approve / Post / Cancel
        - Action visibility

        Output:
        - Proses bisnis hidup

FASE 3 — BACKEND-DRIVEN UI
    Tujuan: backend = sumber kebenaran

    SESSION 8 — UI Schema API
        - Backend menyediakan:
        - entity schema
        - form schema
        - workflow schema

        Frontend:
        - fetch & cache schema
        - render otomatis

        Output:
        - Tambah entity tanpa ubah frontend

    SESSION 9 — Backend Pagination & Filter
        - Server-side filter
        - Query builder
        - Search
        - Advanced filter

        Output:
        - Data besar aman

FASE 4 — EXTENSION SYSTEM (VERTICAL)
    Tujuan: niche tumbuh tanpa merusak core

    SESSION 10 — Vertical Module Loader
        - Register page
        - Register route
        - Register menu

        Output:
        - Vertical = plug-in

    SESSION 11 — Vertical UI Override
        - Custom page
        - Custom form
        - Custom workflow

        Output:
        - Apotek ≠ Bengkel ≠ Klinik

FASE 5 — PLATFORM UX & RELIABILITY
    Tujuan: siap dipakai production

    SESSION 12 — Error & Feedback

        - Loading state

        Output:
        - UX profesional

    SESSION 13 — Performance & Caching
        - Schema cache
        - Data cache

        Output:
        - Cepat & scalable

FASE 6 — ENTERPRISE READINESS
