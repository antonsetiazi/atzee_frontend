1. Prinsip Baru
=============================================
    - Frontend selalu merender dashboard 
        berdasarkan role yang diberikan backend.
    - Frontend tidak peduli siapa usernya
    - Frontend hanya membaca schema UI


2. Flow Baru Platform
=============================================
    User datang
        ↓
    Frontend minta UI Schema
        ↓
    Backend cek session
        ↓
    Jika tidak login → role = guest
    Jika login → role = sesuai user
        ↓
    Backend kirim dashboard schema
        ↓
    Frontend render

    Guest Dashboard ≠ Fake Dashboard
    Guest Dashboard = Dashboard dengan role guest


3. Arsitektur yang Terjadi
=============================================    
    {
        "role": "guest",
        "menus": [
            { "label": "Browse Services", "page": "marketplace" },
            { "label": "Login", "page": "login" },
            { "label": "Register", "page": "register" }
        ]
    }


4. Perubahan yang Dibutuhkan di Router
============================================= 
    <Routes>

        {/* PUBLIC + GUEST */}
        <Route
            path="/"
            element={
                <AppBootstrap>
                    <AppLayout />
                </AppBootstrap>
            }
        >
            {usersRoutes.map(renderRoute)}
            {PageRoutes()}
        </Route>

        <Route path="/login" element={<LoginPage />} />
        <Route path="/register" element={<RegisterPage />} />

    </Routes>

    Guest juga boleh masuk AppLayout


5. Proteksi Berpindah ke Level Action
============================================= 
    Jadi bukan halaman yang dilock.
    Tetapi aksi yang dilock.

    Ini sama seperti:
        - Shopee
        - Gojek


6. Kenapa Ini Sangat Cocok Dengan Atzee      
============================================= 
    Karena platform ATZEE berbasis:
        Backend Driven UI

    Jadi sebenarnya sangat natural untuk punya role tambahan:
        guest

=============================================

                       │
                role detection
                       │

8. Dampak Positif Besar
=============================================
    Dengan ini Atzee otomatis bisa menjadi:

    1️⃣ ERP Platform
        login → dashboard

    2️⃣ Marketplace
        guest → browse
        login → transact
        
    3️⃣ Service Platform
        guest → search service
        login → booking

    Tanpa mengubah frontend core.
